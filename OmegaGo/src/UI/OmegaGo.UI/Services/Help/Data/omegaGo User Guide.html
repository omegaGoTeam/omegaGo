User Guide

 

Go is an ancient game which, from its forgotten origins in China, spread first to the rest of East Asia, and then to the entire world. How ancient? Historians aren't sure; it has a definite history of over 2500 years, but it might possibly be older.

Yet Go endured and is still popular even today. With omegaGo, whether you are a beginner or an advanced player, you may delve into the world of Go and enjoy the fun of its strategic gameplay.

Features

With omegaGo, you may:

·         Play Go against another player on the same device, over the Internet, or against an AI program

·         Learn the basics of Go in a tutorial

·         Keep track of your games and victories

·         Earn points and titles by completing quests

·         Solve life and death puzzles to practice your game

·         Connect to an online server of your choice – Pandanet or KGS

·         View and edit SGF files

·         Learn about Go in an extensive help library

·         Play in Czech or English

System Requirements

Minimum system requirements:

·         OS: Windows 10 Anniversary Update

·         Processor: x86, x64 or ARM

·         Memory: 1 GB RAM

·         Storage: 200 MB

·         A Microsoft account

Note:

·         A desktop PC, a mobile phone and a tablet that satisfy the above requirements will always run omegaGo. XBox One may run the game with limitations.

·         Any device capable of running the Windows 10 operating system can run omegaGo.

·         A more powerful processor and additional RAM memory will allow the AI programs to be faster and stronger.

Installation

The game may normally only be installed via Windows Store. To install it that way, find the game in the Store (search for “omegaGo”) and acquire the game by clicking Get the game.

If you don’t have a Microsoft account or for some other reason don’t want to use the Store, you may install the game via sideloading, although this is a complicated process. Contact the developers at omegagoteam@outlook.com to receive assistance.

About this document

The game can be played on desktop computer, mobile devices and on XBox One.  These device families use different means of interaction, but in this document, for consistency, we’ll use desktop controls (“click a button”, “move mouse over a button”) but equivalent actions do the same thing on other device families.

This document is available only in English and will use English titles on buttons and such.

This document describes omegaGo as a client application. It doesn't explain the rules of Go as a board game, but it does point you to the tutorial and to help pages that do explain them.

Screenshots are all taken in the default theme settings and reflect the game as it was in May 2017. The game may have been updated since then so the screenshots may be a little out of date.


 

Navigating and tabs

omegaGo uses a tabbed interface. Each tab you have open contains a screen. The application first starts with the main menu screen opened in a single tab. You may switch between tabs by clicking them or by using keyboard shortcuts on desktop.

To create a new tab, click the "plus" button in the tab row or press Ctrl+T on desktop. To close a tab, click the "x" button in the tab or press Ctrl+W on desktop. Some actions or events will open new tabs on their own, such as when you receive a match request. Some actions or events may also open that new tab in the foreground, making it active, such as when you start observing a game.

Tabs remember their navigation history. You may click the back arrow button on some screens to go back to the previous screen. You may also right-click the tab and select "Go back". Finally, on desktop, you may press Esc to return to the previous screen. You cannot go back from an ingame screen (you must close it instead). Going back from the main menu screen will close the tab.

Closing the last tab will prompt you whether you want to close the application.

Some events will cause a tab to blink. For example, if a move is made in a game that you're observing, the tab with that game will begin to blink. Blinking will stop when you switch to that tab. If that tab was already open, the blinking won't even start.

The title bar also contains a feedback button. Clicking it will open the Feedback Hub which allows you to submit comments to the developers.

Keyboard shortcuts

On desktop, you may use the following keyboard shortcuts:

·         Press Esc to return to the previous screen.

·         Press Alt+Enter to toggle fullscreen mode.

·         Press Ctrl+T to create a new tab with Main menu open.

·         Press Shift+T and Ctrl+Shift+T to move between open tabs.

·         Press Ctrl+1 to Ctrl+9 to switch to the tab with that index.

·         Press Ctrl+W to close the current tab.

·         KGS will send you to the KGS lobby

There are a few additional buttons in the main menu:

·         The button in the bottom left corner allows you to change the game language (you may switch between Czech and English).

o   The option “Auto” will select a language based on the display language of your operating system

Tutorial

The tutorial will introduce you to the game of Go. You don’t need to know anything about Go to play the tutorial.

To exit the tutorial on desktop, you must use the Esc key. Leaving the tutorial will abandon your progress. But don’t worry, the tutorial isn’t long.

In the tutorial screen, from top to bottom, you can see:

·         Sensei’s instructions. Read them to learn about Go and to know what to do.

·         The game board. The tutorial is explained on a 9x9 game board. You can interact with this board when the sensei allows it.

·         A choice dialog. Sometimes, you can respond to sensei in two different ways. Here you may select your answer.

·         A “Next” button. Sometimes, all you need to progress is to click a button.

To progress in the tutorial, depending on what the sensei says, you must:

·         Select a sentence to speak;

·         Click the button; or

·         Select a position on the game board.

Sometimes, sensei will helpfully highlight a position on the game board for you to select. When that happens, you can only progress by clicking the highlighted tile. Sometimes, however, you will be on your own. If you don’t know where to click, reread the sensei’s instructions, and if worst comes to worst, click randomly?

The tutorial is available in both Czech and English.

At the end of the tutorial, you will be returned to the main menu.

Singleplayer Mode and Quests

The singleplayer menu is a hub screen from where you may move to several other screens.

·         Click Tutorial to go to the start of the tutorial (this is also accessible from the main menu).

·         Click Solve puzzles to go to the tsumego menu screen where you can select tsumego problems to solve.

·         Click Play against AI to start a new local game against an AI program. This has the same effect as when you click "Local game" from the main menu, except that your opponent is preselected as Fuego, the AI program we recommend.

·         Click Statistics to go to the statistics screen.

Points and quests

In the singleplayer menu, you may also view your points total. You start with 0 points. Completing and winning games will earn you points, but you will gain the most points by completing quests. Whenever you gain points, that will be shown in a notification bubble in the top right corner of the window.

 

Points and quests are an optional part of the game. Earning points will not cause any additional features of the game to become available and there is little push for you to earn them. It is only up to you!

By earning points, you may be granted ranks. You start at the rank Omega Novice. You will "rank up" when you have 100 points and be granted the next rank. You may continue earning up until you are granted the rank of Omega Graduate at which point you are probably very skilled and working to win real-life Go tournaments! Your current rank and the points total that you need to acchieve the next rank are also shown in the singleplayer menu.

You may have up to 3 "quests" active at a time. You start with 1 quest. Every midnight, local time, you will receive an additional random quest, even if omegaGo is not running. However, you cannot accumulate more than 3 quests as that would violate the restriction of having at most 3 active quests.

Your quests are shown in the singleplayer menu and contain information about how they can be completed. Clicking the Try this now button in the quest will take you to a screen where that quest can be completed. Successfully completing a quest will earn you points specified in its reward. For some quests, this button is not available. For example, the Traditional quest requires you to play on a 19x19 board but it's up to you whether you want to play against an AI, play a hotseat game or play against online players. The poor button wouldn't know where to send you!

Once per day, you may also replace one of your quests. Replacing a quest will destroy it, but you will immediately receive a different randomly selected quest. A day is the period of time between local midnight and the next local midnight.

All of your currently active quests are guaranteed to be distinct (except for a special hidden quest that may sometimes show up!).

Tsumego

A tsumego problem (also called simply "tsumego" and sometimes "a Go puzzle" or "a life-and-death problem") is a puzzle where you are presented with a Go board state and your objective is to kill an enemy group or to ensure your own group lives. Tsumego problems can be found in print or in computer applications such as this one.

Advanced Go players agree that solving tsumego problems is a good way to improve your strength and skill level.

Accessing problems: From the singleplayer menu, click Solve puzzles to move to the tsumego menu screen where you will see all available problems. A check mark means that you have already solved that problem previously. Select a problem to open it.

Once you open a problem, you will have the following controls on screen:

1.       If the Show possibles moves box is checked, the program will advise you by highlighting possible moves that you can make. Making a move outside of the highlighted positions will always be wrong, but not all highlighted positions are correct, either. Having this box checked will greatly decrease the difficulty of the problems.

2.       The Previous problem and Next problem buttons will cause the current problem to be temporarily abandoned and will open another problem.

3.       The Undo last move button will revert the last move you made. Any moves made automatically by the game after the last move made by you are reverted as well.

4.       After that button, you will see Correct! or Wrong! if you made moves that already guarantee that your objective will succeed or fail. As soon as you can see the Correct! box, the problem is considered solved.

Click positions on the board to make moves. After each of your moves, unless the problem is already over (one way or the other), the game will make a move for your opponent. It will make the best possible response to your move. 

You may solve the same puzzle as many times as you want but only the first time will count towards progress on your quests and only the first success will be counted in your statistics.

If you believe you have found an alternate solution to a puzzle that is not accepted by the app, please send us a message at omegagoteam@outlook.com. Mention the name of the problem please.

The bar on the left contains additional information about tsumego or about the specific problem, but if you don't need that information, you may ignore it. You generally won't need the information.

Creating a game

When you're setting up a game or when you're deciding whether to accept or refuse a game, you will be presented with a screen such as this:

You will note a number of options. Some of them might not be visible or enabled under some circumstances. For example, when you're looking at a KGS challenge, you won't be able to change the board size (you're not the challenge creator) and when you're playing on Pandanet, your time control options will be limited because the server doesn't support all time control systems that omegaGo supports.

·         Board size. Determines the size of the Go board. If you don't want to play on a standard size board, you may click Custom... to use any size you want (with reason; your computer probably won't handle a 100x100 board very well). Servers have strict limitations on permitted board sizes and require the board to be square. Most AI programs also need a square board size.

·         Ruleset. Determines whether the game will use Chinese, Japanese or AGA rules.

·         Black player. Determines whether the black player is a human playing on this device or an AI program and if so, what program. Some AI programs are configurable, see the section "AI programs" for details. To configure an AI program, click the button More....

·         White player. As above, except for white.

·         Switch colors. Pressing this button will exchange the values in the Black player and White player fields.

·         Handicap. Determines the number of black handicap stones that will be placed on the board automatically as the game begins.

·         Time control. Shows the time control system used for the game. Click the button next to it to change it. Several time control systems are available: no time control, absolute timing, japanese byo-yomi and canadian byo-yomi. All systems are explained in the form.

·         Komi. Determines the amount of points that will be added to white's score as compensation for playing second. Komi may be negative. Komi must be a half-integer, such as "0", "2" or "7.5".

·         Submission button. The button at the bottom of the form will validate the form and if the validation succeeds, the game will be created. What exactly happens depends on how you came to the form (see further down).

Validation. You cannot start a game with invalid values. If you entered unsupported values in some of the form fields, you will be  told when you submit the form, and the form will not submit. This will happen, for example, when you put a non-integer value into a field that requires integers.

Methods of creating a game

There are several methods of how a game can come to exist.

1.       Local. A local game may be created from the main menu or from the singleplayer menu. In this case, the submission button will create and open that game in a new tab.

2.       Pandanet Challenge. You may challenge a Pandanet player from the Pandanet lobby. In that case, you will be limited to options that Pandanet supports and instead of selecting an AI (or Human) for each color, you will choose which color you want to play yourself. Your opponent's name will also be displayed. The submission button will issue the challenge to that player and close the form.

3.       Pandanet Incoming Challenge. When a player challenges you on Pandanet, a new blinking tab will appear with that challenge. You will not be able to change any values on that form. There will be two buttons, one to accept and one to reject the challenge. Accepting will create the game and open it in a new tab. Both buttons will close the form.

4.       KGS Setup Challenge. You may create an open KGS challenge in a KGS room. You will be limited to options that KGS supports in much the same way Pandanet challenge works, except that it won't be for a specific opponent. The submission button will close the form and create that challenge. A new tab with that challenge will open immediately and then you will need to wait for a challenger.

5.       KGS Challenge. When you create or join a challenge, you will be shown the challenge's properties but you won't be able to change them. Your opponent's name will be shown. Both players need to accept the challenge for it to be created. When both do, the game will be created and the challenge form will close.

a.       If you joined the challenge of another player, you will be able to accept it (after they accept your challenge) or remove yourself from the challenge.

b.       If you created the challenge, you will be able to accept a challenger as your opponent, cancel the challenge altogether or decline only that particular challenger, but leave the challenge open for other players.

Ingame screen

Depending on the game mode, some controls might not be available.

1 – Go board. Click on intersections to make moves or to remove stones during the Stone Removal Phase.

2 – Control menu. Additional actions can be performed from here.

3 – Side bar. Contains information about the game, chat window and offers actions.

4 – Player information. For each player, their color, name, number of captures and remaining time is displayed. If you move mouse over the small line of text under the remaining time display, a tooltip will show up giving details. This information always reflects the current state of the game, not the game as it existed in the past or while analyzing.

5 – Turn player. Says which player is on turn.

6 – Pass. Click this button to pass.

7 – Undo. Click this button to move one move backwards. All moves after your just-before-last move will be undone, including all moves by an AI opponent or online player.

8 – Chat. Type a line in the box and press Enter or click Send to send a message to your opponent. Chat only appears when playing online.

9 – Timeline. Move left to see board state from the past of the game, or move fully to the right to see the current board state of the game. You cannot make moves when you’re not in the last node – you cannot change history. If you want to do so anyway, use the Undo button.

If the omegaGo window is smaller, you might see a different arrangement of controls, or the side bar might be below the game board rather than to the right. You may scroll up and down.

Gameplay

When it’s your turn to play, click an empty intersection on the board to make a move there. Illegal moves won’t be accepted. You may also click Pass to pass. If you’re playing online, the move won’t be made until the server confirms it. In that case, wait a moment until the placed stone appears.

When both players passed in a row (three passes are needed on Pandanet and in some situations under AGA rules), the game moves to Stone Removal Phase. While in this phase, click stones to mark them as removed (dead). Additionally, when playing on KGS, you may click a removed stone to bring it back in the game. Removing a stone might cause other stones to be removed as well – this depends on the server. In local games, all stones in the same group will be removed.

In this phase, you may use buttons to revert all stone removals (except on KGS), to confirm that you’re satisfied with the removed stones or to return back to the game. Returning back to the game is not possible on Pandanet.

When you confirm that you’re satisfied (or when both players confirm if it’s an online game), the game is scored and the results are shown and the game is over.

Control menu

The following options may be available in the control menu:

1.       Resign. Causes you to lose the game. There is no confirmation window.

2.       Get Hint. Asks your assistant to recommend a move for you. This may take a couple of seconds.

3.       Analyze Mode. Enters the Analyze Mode, where you can plan your moves ahead of time or look at what the game would look like if you made different moves. To exit Analyze mode, click Back to game in the side bar.

4.       Hide tabs. Hides the title bar and the tabs so that there’s more space for the board. They reappear when you click the button again or when you switch to a non-game tab.

5.       Save SGF. Saves the current game tree as an SGF file into the library.

6.       System log. Shows a system log of game events in the side bar. The system log is in English only, and may be incomprehensible to many people.

Analyze mode

In Analyze mode, the timeline is hidden and replaced with a game tree and with analyze tools. Analyze mode functions as an SGF editor. You may add markup, add or remove variations and comment on moves. But you can also plan your moves ahead of time by switching between Analyze mode and game mode during a game.

1 – Stone Placement Tool. Select this to place stones of alternating colors on the board.

2 – Delete Branch. Click this to remove the branch starting with the current move from the game tree. Moves that are part of the game cannot be removed (these are the moves from the beginning of the game, in a straight line, until the last real non-analyze move).

3 – Letter Tool. Add letters to points on the board. You will always add the letter that’s as closest to A alphabetically as possible while not yet being on the board. At most 26 letters can be on the board.

4 – Number Tool. Add numbers to points on the board. This functions as Letter Tool, except that you can place unlimited numbers.

5 – Markup Tools. Select this to place markup symbols (square, triangle, circle and x) to points on the board. Attempting to place a symbol on a point that already has a symbol will instead remove the symbol that’s there.

6 – Back to game. Exits Analyze mode and returns you to the current move of the game. This button is not available if you loaded the game tree from the Library instead of clicking Analyze mode while in a game.

7 – Pass. Add a “Pass” move to the current branch. Use this to switch between colors without making a move of the other color.

8 – Commentary box. Type in comments on the current move.

9 – Game tree. Shows the game tree. Click a move to see that move on the board and edit it. The game moves from left to right. The move that’s the most top left is the first move. The move you’re currently viewing and editing is highlighted.

Chat

In online games, you may chat with your opponent.

If it’s a game you’re playing, you, you opponent and any users who download the game record from the server, will be able to see you chat. In addition, any KGS spectators will be able to see your chat.

If it’s a game you’re spectating, only you and other spectators will be able to see your chat. Players will be able to see your chat only when the game is over, if they download the game record from KGS. This is called “kibitzing”.

Online play

You may play against other players on Internet Go servers. Two servers are supported – Pandanet (also known as IGS) and the KGS Go Server.

Playing against other human players is very fun and you will find suitably strong opponents regardless of your skill level on both supported servers if you are willing to play on the full 19x19 board.

User account required. For both servers, you will need a user account. The sections below describe how to get one.

Rank measures your skill level. Each online server maintains your "rank", a level of skill measured from around "30 kyu" (the weakest skill level) up to "1 kyu" (a very strong level), after which the order reverses and the next rank is called "1 dan" and this goes up to "9 dan" which is the strong rank measured by online servers.

"kyu" and "dan" are words from Japanese. The lowest rank for the Pandanet server is 17 kyu. This means that when you first log in to your Pandanet account and view the Statistics screen, you will note that your Pandanet rank is "17k" ("k" is an abbreviation for "kyu").

Your rank rises and falls automatically as you win and lose games. More information about the rating system may be found on the official websites of both servers.

Logging in

The login screen for each server will ask you for your username and password. It will remember your username for that server, and if you check the Remember password button, it will also remember that. These credentials will be roamed and shared between all of your devices where you have omegaGo installed.

You may also check Login at startup to automatically log in to that server when you launch omegaGo. If that login fails, you will be notified with a bubble notification in the top right corner of the window.

You may be logged in to both servers at the same time.

Disconnecting

If you log out or are disconnected, either because you lost internet connection, the server went offline or because you logged in at another device, all ongoing games will be cancelled and you won't be able to return to them.

Pandanet

Registration

To get a Pandanet account, click the link on the Pandanet login screen. You will be taken to a web page where you can create an account, then you must confirm your email address, and then you can log in to your account in omegaGo.

Lobby

In the lobby, you may find games to observe or challenge players to a game.

At the bottom of the screen, you will find a command bar the following buttons:

·         Logout. You will be logged out of Pandanet.

·         Looking for a game. If you check this, you are more likely to receive match requests from other players. You will find in the Visibility submenu in the bottom right of the screen.

·         Refuse all incoming match requests. If you check this, other Pandanet users will not be able to send you match requests. You will find in the Visibility submenu in the bottom right of the screen.

In the Spectate a game tab, you can see a list of games in progress on Pandanet. Use the Refresh button to download an up-to-date list of games. You may sort the games by various criteria.

Left-click a game to start watching the game in a new tab. If that doesn't happen, the game you selected probably ceased to exist since then. Refresh the list to see it updated.

In the Issue a challenge tab, you can see a list of users currently logged in to Pandanet. Use the Refresh button to download an up-to-date list of logged-in users. You may sort the players by various criteria and filter by name.

Left-click a user to open a form where you can set up a challenge. If the setup fails, that user probably logged off in the meantime. Refresh the list to see it updated.

Notes

Not all Pandanet features are supported. Some Pandanet features, such as automatch or nmatch or rooms are not supported by omegaGo.

KGS

Registration

To get a KGS account, click the link on the KGS login screen. This will let you download the official Java-based client for KGS. Launch it, click Play on KGS, then login as a guest. The name that you enter will be your permanent KGS username.

From the main menu of that client, select User-Register.

Then fill in the form and complete the registration process.

Then you will be able to log in with your account to KGS using omegaGo.

This process may seem arcane, but it is the only way to register a new KGS account. The KGS administrators do have plans to create a registration web form in the future, though, so fingers crossed!

Lobby

In the lobby, you may browse, join and exit rooms. You may also browse and join games and challenges.

Logout. At the bottom of the screen, you may use the Logout button to logout from KGS.

Refreshing. Data in the lobby is not updated automatically on the fly. Use the Refresh button in the bottom right to reload up-to-date data.

In the Games and challenges tab, you may select a game container to view the games and challenges within.

You may then select a game or a challenge in the list below. Left-clicking a game or a challenge will cause you to join it. If it's a game, you will be an observer. If it's a challenge, you will be the challenger.

In the Rooms tab, you may view the list of KGS rooms. Selecting a joined room will give you a description of the room. You may join and unjoin rooms. Joining persists across launches. If you join a room, you will still be a member of that room when you log out. You may create a public challenge inside rooms that you have joined. This is described in the section on creating a game.

Notes

Not all KGS features are supported. Some KGS features, such as audio chat, automatch or simul-games, are not supported by omegaGo. Games that are no supported (such as teaching games) will not even be shown in the interface.

Desynchronization. On rare occasions, such as when your opponent requests an undo while placing handicap stones, or when they return to the main gameplay phase of a game after it was already concluded, a desynchronization error may occur if that feature of KGS is not supported by omegaGo. This is something we want to address in the future.

Library

In the library, you may view game records ("SGF files") of games you saved from omegaGo. You may also import and open SGF files you obtained from elsewhere. omegaGo keeps its library of SGF files in a single folder.

You have these global options in this screen:

·         Load SGF file. This will prompt you to select an SGF file on your device. The file will be imported into omegaGo's library.

·         Refresh. This will reload SGF files from omegaGo's library folder.

·         Open folder in Explorer. This will open File Explorer with omegaGo's library folder open. You may move SGF file from this folder or into this folder. This is useful when you want to perform a mass import or a mass delete/mass export. When you're done, click Refresh to make omegaGo reload the contents of the library folder.

After you select an SGF file from the library, you may perform these actions on it:

·         Open. Opens the file as a game tree that's shown in Analyze Mode in the ingame screen. You won't be able to play it, but all options of an SGF editor will be available.

·         Delete. Asks for a confirmation. If confirmed, the selected file will be permanently deleted from the library.

·         Export. This will copy the selected file to a location you choose on your device.

There are limitations on permitted SGF files. For example, only SGF files that represent a single game tree of a Go game are supported. You will be told if an imported SGF file is not supported.

Statistics

Here you may view the number of games of various types that you played and your number of victories. You will also see your current number of points and your omegaGo rank and your rank on online servers.

Available statistics:

·         Total games played: Number of completed games played on this device

·         Solo games played: Counts only games of a human against an AI.

·         Online games played: Counts only games of a human against a Pandanet or KGS opponent.

·         Total games won: Number of won games played on this device. This excludes all hotseat games.

·         Solo games won: Counts only games of a human against an AI.

·         Online games won: Counts only games of a human against a Pandanet or KGS opponent.

·         Tsumego problems solved: A problem is counted as solved even if you didn’t get it right on the first try.

·         Quests completed: Number of quests that you completed and received points for.

·         Points: Your current points total.

·         omegaGo Rank: The rank associated with your number of points. Increasing your rank has no in-game benefits. But it looks cool!

·         Pandanet rank: The rank of the Pandanet account that last logged in to Pandanet on this device using omegaGo

·         KGS rank: The rank of the KGS account that last logged in to KGS on this device using omegaGo

You may also click Reset all progress which will pop up a confirmation dialog. If you confirm, all statistics will be reset to zero, you will lose all points and omegaGo rank, but you will keep your current quests. This will also delete information on what tsumego problems you have solved.

Pandanet rank and KGS rank sometimes have symbols appended to them. The symbol "?" Means that the server is merely estimating your rank and isn't sure yet until you play more games. The rank "NR" means "not ranked". It will change as you play games.

Pandanet and KGS ranks are updated whenever you log in to those servers using omegaGo.

The statistics are saved on your device and will not migrate to other systems where you use omegaGo.

Here’s how the game can look like with a different style:

·         Display.

·         Fullscreen mode. If checked, the game will occupy the entire screen of a single monitor. This is the “borderless windowed” mode, i.e. the window frame will be hidden, but you can alt-tab as normal. Fullscreen mode setting is not remembered and should be set manually each time. AI programs

omegaGo ships with several AI programs that you may play against. Each program has some limitations on what games it can play. Most programs, for example, only support square boards. Each program is described, along with its limitations, when you select it in the Settings menu or when you're creating a local game and click the More button.

There are the following programs:

·         Defeatist. It resigns immediately.

·         Random. Makes random moves.

·         Fish. Also makes random moves, but differently.

·         Puppy. Uses a simple heuristic to select the best move.

·         Fluffy. Uses the minimax algorithm to select the best move.

·         Fuego. An advanced Monte Carlo AI program.

For most users, there will be no reason to play against any program other than Fuego. Fuego is the most fully-featured program in the game and its difficulty is configurable. We recommend you use other programs only if Fuego does not support your game configuration.

Fluffy is configurable. You may set the depth of its minimax tree. We recommend a depth of about 3 for a fast yet intelligent opponent.

Fuego is configurable. You may set three values:

·         Ponder. Whether Fuego should think during its opponent's turn. We recommend leaving this on.

·         Allow resign. Whether Fuego is permitted to resign when the situation appears hopeless (its win percentage drops below a certain value).

·         Playouts. The number of games Fuego will try playing before giving a result. A greater number means more thinking time but also a better move. If you set a very large number, such as several million, Fuego may take up to ten seconds to make a move.

AI programs can be your opponents or assistants. If Fuego is your assistant, it will additionally try to determine what stones should be removed at the end of a local game and automatically mark them removed. You may override its decision, however.

You may ask your assistant to tell you where you should make a move or whether you should resign by using the Get Hint button in the ingame screen's control menu. This button may be disabled depending on your settings.

Two AI programs may play against each other, in which case you're only a spectator. You will, however, perform the Stone Removal Phase yourself (although if Fuego is your assistant, it will help).

Troubleshooting

These answers may help you when something doesn’t work right:

1.       I don’t know how to play online.

a.       To play online on Pandanet, you must first create an account here:
http://pandanet-igs.com/igs_users/register
Then, you must confirm it by clicking on the link in your mailbox, and only then you may use your username and password to login using omegaGo.

2.       The game crashed. What to do?

a.       If you can easily reproduce the crash (for example, it happens every time you perform a specific action), please tell us about it at omegagoteam@outlook.com and we’ll fix it as soon as possible. Until then, avoid that action if possible.

b.       If it seems to happen randomly, but it only happens while you’re playing Fuego, then it’s probably because of a bug in the AI program. There’s little that can be done about that. Endure the crashes, please ^^.

c.       Otherwise, start the game again and hope it doesn’t reoccur. If automatic crash reporting is enabled in your system (it probably is), then we’ll know about the crash and we’ll try to figure out what’s wrong.

3.       I am confused by the rules.

a.       First, if you haven't already, try playing the tutorial.

b.       Second, read the "Introduction to Go" page in the Help screen.

c.       If that doesn't help, you're probably confused either by the concept of "two eyes" or by the stone removal phase (we also call it life/death determination phase). If you're a technical type, try reading "Comprehensive Rules of Go" in the Help screen. Otherwise, you may try solving a couple of tsumego problems. If all else fails, find a human teacher. There is a Go club in most cities and Go players are usually very enthusiastic about teaching!
